# Quiz Creation Guide for AI Agents

This document describes how to create new quizzes for the MyGrammar Quiz app. The app is used by KangChiao elementary students (Grade 3-4) in Taiwan.

## Folder Structure

Quizzes are organized in a two-level folder hierarchy under `public/quizzes/`:

```
public/quizzes/
  MyGrammar/              <-- Book (top-level folder)
    Unit_1/               <-- Unit (sub-folder)
      plural.json         <-- Quiz file
      nouns.json
    Unit_2/
      verbs.json
  MyView/                 <-- Another book
    Test_1/
      sample.json
```

- **Book** = top-level folder (shown on the landing page)
- **Unit** = sub-folder within a book (shown when you tap a book)
- **Quiz** = a `.json` file within a unit (shown when you tap a unit)

## Steps to Add a New Quiz

1. Create the folder path if it doesn't exist: `public/quizzes/<Book>/<Unit>/`
2. Create your quiz `.json` file inside that folder (see `quiz-sample.json` for a complete example).
3. Run `npm run generate-index` (or it runs automatically on `npm start` / `npm run build`).
4. That's it -- the app discovers quizzes from the folder structure automatically.

**You do NOT need to edit `index.json` manually.** It is auto-generated by `scripts/generate-quiz-index.js`.

## Naming Conventions

- **Book folders**: Use PascalCase or descriptive names (e.g., `MyGrammar`, `MyView`)
- **Unit folders**: Use underscores for spaces (e.g., `Unit_1`, `Chapter_3`). Underscores are displayed as spaces in the UI.
- **Quiz files**: Use lowercase with underscores (e.g., `plural.json`, `common_nouns.json`)

## Quiz File Structure

Each quiz file has three sections:

### `meta` -- Quiz Metadata

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Must match the path: `<Book>/<Unit>/<filename>` (without `.json`) |
| `version` | string | Always `"1.0"` |
| `title` | string | Display title, e.g. `"Unit 3 — Action Verbs"` |
| `subject` | string | Always `"English Grammar"` |
| `level` | string | Always `"Elementary Grade 3–4"` |
| `created` | string | ISO date `YYYY-MM-DD` |
| `description` | string | One-line summary of the quiz topic |

### `settings` -- Quiz Behavior

| Field | Type | Description |
|-------|------|-------------|
| `shuffle` | boolean | Randomize question order (usually `true`) |
| `maxQuestions` | number \| null | Limit questions shown, or `null` for all |

### `questions` -- Array of Question Objects

Each question has these common fields:

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Sequential: `"q1"`, `"q2"`, `"q3"`, ... |
| `type` | string | One of the 5 types below |
| `difficulty` | 1 \| 2 \| 3 | 1 = easy, 2 = medium, 3 = hard |
| `tags` | string[] | Lowercase, hyphen-separated keywords |
| `question` | string | The question text shown to the student |
| `explanation` | string | Shown after answering, should be educational |

## Question Types

### 1. `multiple-choice`

Student picks one correct answer from a list.

| Field | Type | Description |
|-------|------|-------------|
| `options` | string[] | 3-4 answer choices |
| `answer` | number | **Zero-based index** of the correct option |

```json
{
  "id": "q1",
  "type": "multiple-choice",
  "difficulty": 1,
  "tags": ["verbs", "action"],
  "question": "Which word is an action verb?",
  "options": ["happy", "run", "blue", "tall"],
  "answer": 1,
  "explanation": "\"run\" is an action verb — it describes something you do."
}
```

**Tips:** Distractors should be plausible but clearly wrong. Avoid "all of the above" or trick answers.

---

### 2. `true-false`

Student decides if a statement is true or false.

| Field | Type | Description |
|-------|------|-------------|
| `answer` | boolean | `true` or `false` |

```json
{
  "id": "q2",
  "type": "true-false",
  "difficulty": 1,
  "tags": ["verbs", "identification"],
  "question": "\"Jump\" is an action verb.",
  "answer": true,
  "explanation": "Yes! \"Jump\" describes an action you can do."
}
```

**Tips:** Statement should be unambiguous. Avoid double negatives.

---

### 3. `fill-in`

Student types the answer. Supports multiple accepted spellings/capitalizations.

| Field | Type | Description |
|-------|------|-------------|
| `answer` | string | The primary correct answer |
| `acceptedAnswers` | string[] | All accepted variants (include capitalized forms) |
| `hint` | string (optional) | Hint shown to the student |

```json
{
  "id": "q3",
  "type": "fill-in",
  "difficulty": 2,
  "tags": ["verbs", "past-tense"],
  "question": "Yesterday, I ___ (walk) to school.",
  "answer": "walked",
  "acceptedAnswers": ["walked", "Walked"],
  "hint": "Add -ed for regular past tense",
  "explanation": "walk → walked (regular verb, add -ed for past tense)"
}
```

**Tips:** Always include at least the capitalized variant. Keep expected answers to one word.

---

### 4. `categorize`

Student drags/sorts items into category buckets.

| Field | Type | Description |
|-------|------|-------------|
| `items` | string[] | All words to be sorted (4-6 items) |
| `categories` | array | 2-3 buckets, each with `label` and `answers` |

```json
{
  "id": "q4",
  "type": "categorize",
  "difficulty": 2,
  "tags": ["verbs", "nouns", "sorting"],
  "question": "Sort each word — is it a verb or a noun?",
  "items": ["run", "apple", "swim", "table", "sing"],
  "categories": [
    { "label": "Verb", "answers": ["run", "swim", "sing"] },
    { "label": "Noun", "answers": ["apple", "table"] }
  ],
  "explanation": "run/swim/sing are actions (verbs). apple/table are things (nouns)."
}
```

**Tips:** Every item must appear in exactly one category. Use 4-6 items for young learners.

---

### 5. `reorder`

Student arranges scrambled words into the correct sentence.

| Field | Type | Description |
|-------|------|-------------|
| `words` | string[] | Scrambled word list |
| `answer` | string[] | Words in correct order |

```json
{
  "id": "q5",
  "type": "reorder",
  "difficulty": 3,
  "tags": ["sentence", "word-order"],
  "question": "Arrange the words to form a correct sentence:",
  "words": ["school", "to", "walks", "He"],
  "answer": ["He", "walks", "to", "school"],
  "explanation": "Correct sentence: \"He walks to school.\""
}
```

**Tips:** Keep sentences to 4-6 words. First word in answer should be capitalized.

---

## Rules Summary

- **10-20 questions** per quiz, using **at least 3 different question types**
- Start with easier questions (difficulty 1), increase gradually
- Use **Grade 3-4 appropriate vocabulary**
- Explanations should be clear and teach the concept, not just state the answer
- Tags are **lowercase** and **hyphen-separated**
- `meta.id` must match the path: `<Book>/<Unit>/<filename>` (without `.json`)
- `index.json` is **auto-generated** -- never edit it manually
